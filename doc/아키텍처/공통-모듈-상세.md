# 공통 모듈 상세

`com.forma.common` 패키지는 프레임워크의 핵심이며, 도메인 코드가 상속/사용하는 베이스 클래스와 유틸리티를 제공한다.

## 1. BaseController

**위치**: `common/base/BaseController.java`

추상 클래스로, 모든 도메인 Controller가 상속한다. `service()` 하나만 구현하면 CRUD API가 자동 제공된다.

| 메서드 | HTTP | 경로 | 설명 |
|--------|------|------|------|
| `list()` | GET | `/api/{entity}` | 검색 + 페이징 목록 |
| `get()` | GET | `/api/{entity}/{id}` | 단건 조회 |
| `save()` | POST | `/api/{entity}` | 저장 (INSERT or UPDATE) |
| `delete()` | DELETE | `/api/{entity}/{id}` | 삭제 |

도메인 Controller는 master-detail의 `/save-all`, 상태 변경의 `/approve` 등 커스텀 엔드포인트만 추가한다.

## 2. BaseService

**위치**: `common/base/BaseService.java`

모든 도메인 Service가 상속하는 추상 클래스. 4개 추상 메서드를 구현하면 CRUD가 완성된다.

### 필수 구현 메서드

| 메서드 | 반환 | 예시 |
|--------|------|------|
| `jdbc()` | JdbcTemplate | 주입받은 JdbcTemplate 반환 |
| `table()` | String | `"tb_customer"` |
| `pk()` | String | `"cust_cd"` |
| `selectColumns()` | String | `"t.*"` 또는 JOIN 포함 컬럼 |

### 선택 오버라이드 메서드

| 메서드 | 기본값 | 설명 |
|--------|--------|------|
| `appendWhere()` | 조건 없음 | 검색조건 WHERE절 조립 |
| `defaultSort()` | `pk() + " DESC"` | 기본 정렬 |

### 주요 기능

**list() - 목록 조회**
- 예약 파라미터: `_page`(기본 1), `_size`(기본 50), `_sort`(기본 defaultSort())
- COUNT 쿼리 → SELECT 쿼리 → `PageResult` 반환
- `appendWhere()`로 검색조건을 동적 추가

**save() - INSERT/UPDATE 자동 판별**
- PK 값이 없거나 DB에 없으면 → `doInsert()`
- PK 값이 있고 DB에 있으면 → `doUpdate()`
- Map의 key-value를 동적으로 SQL 조립

**replaceChildren() - 마스터-디테일 저장**
- 자식 테이블의 기존 데이터를 FK 기준으로 전삭
- 새 데이터를 순서대로 INSERT (seq 필드 자동 채번)
- 호출 예: `replaceChildren("tb_sales_order_item", "order_no", orderNo, items)`

## 3. ApiResponse

**위치**: `common/model/ApiResponse.java`

모든 API의 응답 래퍼. JSON 직렬화 구조:

```json
// 성공
{ "success": true, "data": { ... } }

// 실패
{ "success": false, "message": "에러 메시지" }
```

팩토리 메서드:
- `ApiResponse.ok(data)` - 데이터 포함 성공
- `ApiResponse.ok()` - 데이터 없는 성공 (void)
- `ApiResponse.error("메시지")` - 실패

## 4. PageResult

**위치**: `common/model/PageResult.java`

목록 조회의 페이징 결과. JSON 직렬화 구조:

```json
{ "data": [...], "total": 42, "page": 1, "size": 50 }
```

## 5. SeqGenerator

**위치**: `common/util/SeqGenerator.java`

PREFIX-날짜-순번 형태의 채번기. `forma_seq` 테이블을 자동 생성하여 관리한다.

- 형식: `{PREFIX}-{yyyyMMdd}-{3자리순번}`
- 예시: `ORD-20260317-001`, `ORD-20260317-002`
- 호출: `seqGenerator.next("ORD")`
- 날짜가 바뀌면 순번이 1부터 다시 시작

## 6. CodeController

**위치**: `common/util/CodeController.java`

코드 사전 API. `codes.json` 파일을 기동 시 로딩하여 서비스한다.

| API | 설명 |
|-----|------|
| `GET /api/codes` | 전체 코드 사전 |
| `GET /api/codes/{group}` | 특정 코드 그룹 (예: ORD_STATUS) |

프론트엔드의 combo, badge 위젯이 이 API로 코드 목록을 가져간다.
`getCodeItems()` 함수가 클라이언트 측 캐싱을 담당한다.

### codes.json 구조

```json
{
  "ORD_STATUS": {
    "label": "수주상태",
    "items": [
      {"value": "DRAFT", "label": "작성중", "color": "#888"},
      {"value": "CONFIRMED", "label": "확정", "color": "#378ADD"}
    ]
  }
}
```

## 7. BusinessException

**위치**: `common/util/BusinessException.java`

비즈니스 예외. `GlobalExceptionHandler`가 캐치하여 `ApiResponse.error(message)` 형태로 반환한다.

```java
throw new BusinessException("작성중 상태에서만 승인 가능합니다");
// → { "success": false, "message": "작성중 상태에서만 승인 가능합니다" }
```

## 8. WebConfig / GlobalExceptionHandler

**위치**: `common/config/WebConfig.java`

- `WebConfig`: CORS 설정 (`/api/**` 전체 허용)
- `GlobalExceptionHandler`:
  - `BusinessException` → `ApiResponse.error(e.getMessage())`
  - 기타 `Exception` → 로그 출력 + `ApiResponse.error("시스템 오류: ...")`
