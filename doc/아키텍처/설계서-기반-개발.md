# 설계서 기반 개발 프로세스

## 개요

FORMA의 핵심 워크플로는 **YAML 설계서 → AI 코드 생성**이다.
설계서는 `design/` 디렉토리에 위치하며, AI에게 컨텍스트로 전달되어 코드를 생성한다.

## 설계서 파일 구조

```
design/
├── _entities.yml        # 전체 엔티티 정의 (필드, 타입, 관계)
├── _schema_guide.yml    # 설계서 YAML 스키마 가이드
├── screens/             # 화면별 설계서
│   └── {화면ID}.yml     #   CUS-001.yml, ORD-001.yml
└── _prompts/            # AI 프롬프트 템플릿
    ├── 01-generate-design.md   # 요구사항 → 설계서 변환
    └── 02-generate-code.md     # 설계서 → 코드 생성
```

## 엔티티 정의 (_entities.yml)

모든 엔티티의 단일 소스(Single Source of Truth).

```yaml
customer:
  table: tb_customer        # SQL 테이블명
  label: 거래처              # 표시명
  pk: cust_cd               # 기본키
  fields:
    cust_cd:   { type: string, label: 거래처코드, len: 20 }
    cust_type: { type: code, label: 거래처유형, code: CUST_TYPE }
  audit: true               # created_by/at, updated_by/at 자동 추가

sales_order:
  children:                 # 마스터-디테일 관계
    - entity: sales_order_item
      fk: order_no
```

### 필드 타입

| type | 설명 | 추가 속성 |
|------|------|-----------|
| `string` | 문자열 | `len`: 길이 |
| `number` | 정수 | - |
| `decimal` | 소수점 | `p`: precision, `s`: scale |
| `date` | 날짜 | - |
| `text` | 장문 텍스트 | - |
| `code` | 코드 사전 참조 | `code`: 코드그룹명 |

### 특수 속성

| 속성 | 설명 | 예시 |
|------|------|------|
| `gen` | 자동 채번 | `PREFIX_SEQ:ORD` → SeqGenerator 사용 |
| `ref` | 외래키 참조 | `customer.cust_cd` |
| `from` | lookup 필드 | `"cust_cd→customer.cust_nm"` |
| `calc` | 계산 필드 | `"qty * unit_price"` |
| `default` | 기본값 | `DRAFT` |

## 화면 설계서 (screens/{화면ID}.yml)

### 화면 유형

| type | 설명 | 구성 |
|------|------|------|
| `list` | 검색 + 그리드 목록 | search + master |
| `master-detail` | 전표형 (상위+하위 그리드) | search + master + detail |
| `form` | 등록/수정 폼 | search + form |
| `tab-form` | 탭별 폼/그리드 | tabs |
| `tree-grid` | 좌측 트리 + 우측 그리드 | tree + master |

### 설계서 구조 예시 (CUS-001.yml)

```yaml
screen:
  id: CUS-001
  name: 거래처 관리
  type: list
  entity: customer
  menu: 기준정보 > 거래처관리
  roles: [ROLE_USER]

search:                    # 검색조건 → appendWhere()로 변환
  - field: cust_cd
    widget: text           # LIKE '%value%'
  - field: cust_type
    widget: combo          # = ?
    code: CUST_TYPE

master:                    # 그리드 컬럼 → FormaGrid columns로 변환
  columns:
    - field: cust_cd
      width: 120
      frozen: true
    - field: cust_type
      format: badge
      code: CUST_TYPE
  defaultSort: cust_cd ASC
  features: [export]

actions:                   # 버튼 → FormaPage.actionBar로 변환
  - type: SEARCH
  - type: SAVE
  - type: DELETE
    confirm: true

notes: |                   # 자연어 지시 → 구현에 반영
  - 그리드에서 직접 수정 가능 (inline edit)
```

### 검색 위젯 → appendWhere 변환 규칙

| widget | SQL 조건 | 파라미터명 |
|--------|----------|------------|
| `text` | `LIKE '%value%'` | `{field}` |
| `combo` | `= ?` | `{field}` |
| `combo` (multi) | `IN (?, ?, ...)` | `{field}` (콤마 구분) |
| `dateRange` | `>= ?` AND `<= ?` | `{field}_from`, `{field}_to` |
| `codePopup` | `= ?` | `{field}` |

### 설계서 rules 섹션

```yaml
rules:
  - trigger: [qty, unit_price]     # 셀 변경 시 자동 계산
    calc: "qty * unit_price → amount"
  - validate: "qty > 0"           # 유효성 검증
    message: "수량은 0보다 커야 합니다"
  - state: status                  # 상태 전이 규칙
    flow:
      DRAFT: [CONFIRMED]          # DRAFT에서 CONFIRMED으로만 변경 가능
```

## 코드 생성 워크플로

### 1단계: 요구사항 → 설계서 생성
- `design/_prompts/01-generate-design.md` 프롬프트 사용
- PL의 자연어 요구사항을 YAML 설계서로 변환
- `_entities.yml`에 엔티티가 없으면 추가

### 2단계: 설계서 → 코드 생성
- `design/_prompts/02-generate-code.md` 프롬프트 사용
- 설계서 + `_entities.yml`을 입력으로 다음 파일 생성:

| 생성 파일 | 경로 |
|-----------|------|
| Controller | `src/main/java/com/forma/domain/{모듈}/{기능}/{Entity}Controller.java` |
| Service | `src/main/java/com/forma/domain/{모듈}/{기능}/{Entity}Service.java` |
| HTML | `src/main/resources/static/pages/{모듈}/{화면ID}.html` |
| JS | `src/main/resources/static/pages/{모듈}/{화면ID}.js` |
| DDL | `src/main/resources/schema/{entity}.sql` |

### 3단계: 개발자 검토 및 수정
- 생성된 코드를 검토하고 비즈니스 로직 보완
- `notes` 섹션의 요구사항이 구현에 반영되었는지 확인
