# FORMA 시스템 아키텍처 개요

## 1. 프로젝트 목적

FORMA는 ERP/업무시스템을 빠르게 구축하기 위한 AI 활용 개발 프레임워크이다.
**설계서(YAML) → AI 코드 생성 → 개발자 수정**의 3단계로 화면을 개발한다.

## 2. 전체 시스템 구조

│                    브라우저                            │

## 4. 요청-응답 흐름

### 4.1 목록 조회 (GET /api/customer?cust_nm=삼성&_page=1&_size=50)

```
1. 브라우저: FormaApi.get('/api/customer', search.getValues())
2. BaseController.list() → params를 Map<String, String>으로 수신
3. BaseService.list():
   a. _page, _size, _sort 파라미터 추출
   b. COUNT 쿼리 조립: "SELECT COUNT(*) FROM tb_customer t WHERE 1=1"
   c. appendWhere()로 검색조건 추가: "AND t.cust_nm LIKE ?"
   d. SELECT 쿼리 조립 + ORDER BY + LIMIT/OFFSET
   e. PageResult 반환: { data: [...], total: 42, page: 1, size: 50 }
4. ApiResponse.ok(pageResult) → { success: true, data: { data: [...], total: 42 ... } }
5. 브라우저: grid.setData(data.data, data.total)
```

### 4.2 저장 (POST /api/customer)

```
1. 브라우저: FormaApi.post('/api/customer', rowData)
2. BaseController.save() → body를 Map<String, Object>로 수신
3. BaseService.save():
   a. PK값 확인 → 존재 여부로 INSERT/UPDATE 판별
   b. doInsert(): Map의 key-value를 동적으로 INSERT SQL 조립
   c. doUpdate(): PK 제외한 필드로 UPDATE SQL 조립
4. ApiResponse.ok({ success: true, id: "C001" })
```

### 4.3 마스터-디테일 저장 (POST /api/sales-order/save-all)

```
1. 브라우저: FormaApi.post('/api/sales-order/save-all', { master: {...}, items: [...] })
2. SalesOrderController.saveAll() → SalesOrderService.saveAll()
3. Service:
   a. PK 없으면 SeqGenerator로 채번 (ORD-20260317-001)
   b. save()로 마스터 저장
   c. items의 각 행에 계산 로직 적용 (amount = qty * unit_price)
   d. replaceChildren()으로 기존 디테일 삭제 후 재입력
   e. 합계 갱신 (total_amt = SUM(amount))
```

## 5. 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| **Map 기반 데이터 전달** | Entity POJO 대신 `Map<String, Object>`를 전 계층에서 사용. 설계서 변경 시 Java 코드 수정 최소화 |
| **상속 기반 CRUD** | BaseController/BaseService 상속만으로 4개 API 자동 완성. 도메인 코드는 차이점만 구현 |
| **동적 SQL 조립** | SQL을 StringBuilder로 동적 조립. 정적 매핑(JPA/MyBatis) 없이 유연한 쿼리 가능 |
| **프론트 빌드 제로** | Vanilla JS + 정적 파일 서빙. 빌드 도구 없이 HTML/JS 직접 배치 |
| **설계서 주도 개발** | YAML 설계서가 코드 생성의 단일 소스. 엔티티/화면/코드 사전 모두 설계서에서 관리 |
