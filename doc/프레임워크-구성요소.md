# FORMA v2 프레임워크 구성요소

## JS 컴포넌트 (14개 파일)

| 파일 | 역할 |
|------|------|
| forma.core.js | platform.post/get (Promise/async), Listener, RESULT_CODE |
| forma.grid.js | 그리드 (에디터8종, 셀병합, 트리, 마스터-디테일, 가상스크롤, 그룹핑, 키보드, 클립보드, Undo, 필터행, 렌더러, 드래그이동, XLSX/CSV/인쇄) |
| forma.form.js | 폼 (위젯 21종, isDirty, destroy, 위젯 패턴 아키텍처) |
| forma.toolbar.js | 권한 기반 버튼바 (F3~F12 단축키) |
| forma.tab.js | 탭 전환 (hideTab/showTab) |
| forma.modal.js | 모달 팝업 (중첩 지원, 드래그, 부모↔자식 통신) |
| forma.popup.js | Alert/Confirm (Promise)/Loading/Toast |
| forma.util.js | 유틸리티 40+개 (날짜/숫자/문자열/배열/검증/코드캐시) + Split 패널 |
| forma.mdi.js | SPA MDI (iframe 없음, div 기반 탭 관리, 메모리 효율) |
| forma.menu.js | 좌측 트리 메뉴 (FormaMdi.open 연동) |
| forma.i18n.js | 다국어 (ko/en/ja/zh) |
| forma.theme.js | 테마 (light/dark/blue, CSS 변수 기반) |
| forma.chart.js | SVG 차트 5종 (bar/line/pie/donut/hbar) |

## 핵심 인프라

| 기능 | 구현 |
|------|------|
| 인증 | JWT (HS256, 24h) + HttpOnly 쿠키 + TokenInterceptor |
| 메뉴 권한 | tb_role_menu → PgmController → FormaToolbar 버튼 제어 |
| 데이터 권한 | tb_data_auth → DataAuthService → FormaSqlSession 자동 주입 |
| 감사 로그 | AuditLogAspect → save*/delete* 메서드 자동 감지 → tb_audit_log |
| 낙관적 잠금 | updated_at 기반 동시 수정 감지 |
| 파일 첨부 | /api/file/* (업로드/다운로드/삭제) + tb_file |
| MDI | SPA 방식 (iframe 없음), FormaMdi.open() |

## YAML 런타임 엔진

YAML 1파일로 코드 없이 화면 생성:

```
design/screens/MMA010.yml → 품목관리 (list)
design/screens/SDA010.yml → 거래처관리 (list)
design/screens/SDA020.yml → 견적템플릿관리 (split-detail)
design/screens/SOA010.yml → 수주관리 (master-detail + JOIN)
design/screens/SDB010.yml → 수주등록 (split-tab)
```

기존 4파일 패턴(Controller + Service + Mapper + HTML)과 공존.

## 화면 흐름

```
login.html → (JWT 쿠키) → main.html
  ↓
좌측 메뉴 (FormaMenu, API: /api/pgm/menus)
  ↓
메뉴 클릭 → FormaMdi.open()
  ↓
YAML 화면: /api/screen/{id}/definition → 동적 렌더링
커스텀 화면: HTML fetch → script 실행
```
