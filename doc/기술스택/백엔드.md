## 데이터 액세스 패턴

### JdbcTemplate 직접 사용

ORM 없이 `JdbcTemplate`으로 SQL을 직접 작성한다.
데이터는 `Map<String, Object>`로 전달되며, Entity POJO는 참조용으로만 사용한다.

```java
// 목록 조회 - queryForList
List<Map<String, Object>> rows = jdbc.queryForList(sql, params);

// 단건 조회 - queryForObject
Integer count = jdbc.queryForObject("SELECT COUNT(*) ...", Integer.class, id);

// 수정 - update
jdbc.update("UPDATE tb_customer SET cust_nm = ? WHERE cust_cd = ?", name, code);
```

### 동적 SQL 조립

StringBuilder로 SQL을 동적 조립하며, `WHERE 1=1` 패턴을 사용한다.

```java
StringBuilder sql = new StringBuilder("SELECT t.* FROM tb_customer t WHERE 1=1");
List<Object> params = new ArrayList<>();

if (custNm != null && !custNm.isBlank()) {
    sql.append(" AND t.cust_nm LIKE ?");
    params.add("%" + custNm + "%");
}

jdbc.queryForList(sql.toString(), params.toArray());
```

## 데이터베이스

### 개발 환경: H2 (인메모리)

```yaml
# application.yml
spring:
  datasource:
    url: jdbc:h2:mem:forma;DB_CLOSE_DELAY=-1
    driver-class-name: org.h2.Driver
    username: sa
    password:
  h2:
    console:
      enabled: true
      path: /h2
```

- H2 콘솔: http://localhost:8080/h2
- JDBC URL: `jdbc:h2:mem:forma`
- 사용자: `sa`, 비밀번호: 없음

### DDL 자동 실행

`schema.sql`이 애플리케이션 기동 시 자동 실행된다.

```yaml
spring:
  sql:
    init:
      mode: always
```

### 운영 환경: PostgreSQL

운영 배포 시 PostgreSQL로 전환. `application-prod.yml`에서 datasource 변경.

## 테이블 설계 규칙

| 규칙 | 예시 |
|------|------|
| 테이블명: `tb_{snake_case}` | `tb_sales_order` |
| 컬럼명: `snake_case` | `order_no`, `cust_cd` |
| PK: 문자열(채번) 또는 복합키 | `order_no VARCHAR(20)` |
| audit 컬럼 | `created_by`, `created_at`, `updated_by`, `updated_at` |
| `CREATE TABLE IF NOT EXISTS` | 멱등성 보장 |

```sql
CREATE TABLE IF NOT EXISTS tb_customer (
    cust_cd    VARCHAR(20)  NOT NULL PRIMARY KEY,
    cust_nm    VARCHAR(100) NOT NULL,
    cust_type  VARCHAR(50),
    created_by VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_by VARCHAR(50),
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
