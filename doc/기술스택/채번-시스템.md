# 채번(Sequence) 시스템

## 개요

SeqGenerator는 `PREFIX-yyyyMMdd-NNN` 형식의 고유 번호를 생성한다.
전표번호(수주번호, 발주번호 등) 채번에 사용된다.

## 번호 형식

```
{PREFIX}-{날짜8자리}-{3자리순번}
```

예시:
- `ORD-20260317-001` (수주번호, 첫 번째)
- `ORD-20260317-002` (수주번호, 두 번째)
- `PUR-20260317-001` (발주번호, 첫 번째)

날짜가 바뀌면 순번이 `001`부터 다시 시작한다.

## 사용 방법

### Service에서 호출

```java
@Service
public class SalesOrderService extends BaseService {

    private final SeqGenerator seq;

    public SalesOrderService(JdbcTemplate jdbc, SeqGenerator seq) {
        this.jdbc = jdbc;
        this.seq = seq;
    }

    @Transactional
    public Map<String, Object> saveAll(Map<String, Object> body) {
        Map<String, Object> master = (Map<String, Object>) body.get("master");

        String orderNo = (String) master.get("order_no");
        if (orderNo == null || orderNo.isBlank()) {
            orderNo = seq.next("ORD");  // ORD-20260317-001
            master.put("order_no", orderNo);
        }
        // ...
    }
}
```

### 설계서에서 지정

```yaml
# _entities.yml
sales_order:
  fields:
    order_no: { type: string, label: 수주번호, len: 20, gen: PREFIX_SEQ:ORD }
```

`gen: PREFIX_SEQ:{PREFIX}` 속성이 있으면 코드 생성 시 SeqGenerator 사용.

## 내부 구조

### forma_seq 테이블

SeqGenerator가 초기화 시 자동으로 생성하는 채번 관리 테이블.

```sql
CREATE TABLE IF NOT EXISTS forma_seq (
    prefix    VARCHAR(20) NOT NULL,
    date_part VARCHAR(8)  NOT NULL,
    last_seq  INT         NOT NULL DEFAULT 0,
    PRIMARY KEY (prefix, date_part)
);
```

### 동작 원리

1. `UPDATE forma_seq SET last_seq = last_seq + 1` 시도
2. 해당 날짜의 레코드가 없으면 `INSERT ... last_seq = 1`
3. `SELECT last_seq`로 현재 값 조회
4. `PREFIX-datePart-NNN` 형식으로 반환
