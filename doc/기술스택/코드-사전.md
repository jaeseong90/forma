# 코드 사전 시스템

## 개요

코드 사전은 콤보박스, 배지(badge) 등 UI에서 코드값을 표시명으로 변환하는 시스템이다.
`codes.json` 파일에 정의되며, `CodeController`가 API로 서비스한다.

## 구조

### codes.json

```json
{
  "코드그룹명": {
    "label": "그룹 표시명",
    "items": [
      { "value": "코드값", "label": "표시명", "color": "#색상코드" }
    ]
  }
}
```

### 현재 등록된 코드 그룹

| 그룹 | 라벨 | 항목 |
|------|------|------|
| `ORD_STATUS` | 수주상태 | DRAFT(작성중), CONFIRMED(확정), APPROVED(승인), REJECTED(반려), CLOSED(마감) |
| `CUST_TYPE` | 거래처유형 | DOMESTIC(국내), OVERSEAS(해외), INTERNAL(내부) |

## API

| 엔드포인트 | 설명 |
|-----------|------|
| `GET /api/codes` | 전체 코드 사전 |
| `GET /api/codes/{codeGroup}` | 특정 코드 그룹 |

### 응답 예시

```json
// GET /api/codes/ORD_STATUS
{
  "success": true,
  "data": {
    "label": "수주상태",
    "items": [
      {"value": "DRAFT", "label": "작성중", "color": "#888"},
      {"value": "CONFIRMED", "label": "확정", "color": "#378ADD"}
    ]
  }
}
```

## 프론트엔드 연동

### 검색바 콤보

```javascript
{ field: 'cust_type', widget: 'combo', label: '유형', code: 'CUST_TYPE' }
// → FormaSearch가 getCodeItems('CUST_TYPE')를 호출하여 <option> 자동 생성
```

### 그리드 배지

```javascript
{ field: 'status', format: 'badge', code: 'ORD_STATUS' }
// → FormaGrid가 코드의 color 속성으로 색상 배지 렌더링
```

### 캐싱

`getCodeItems(codeGroup)` 함수가 최초 1회만 API 호출하고 이후 캐시에서 반환한다.

## 새 코드 그룹 추가 방법

1. `src/main/resources/codes.json`에 그룹 추가
2. `design/_entities.yml`의 필드에 `code: 새그룹명` 지정
3. 화면 설계서에서 `widget: combo, code: 새그룹명` 또는 `format: badge, code: 새그룹명` 사용

### 네이밍 규칙

코드 그룹명: `UPPER_SNAKE_CASE` (예: `ORD_STATUS`, `CUST_TYPE`)
코드값: `UPPER_SNAKE_CASE` (예: `DRAFT`, `CONFIRMED`, `DOMESTIC`)
