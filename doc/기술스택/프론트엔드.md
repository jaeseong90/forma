# 프론트엔드 기술 스택

## 핵심 기술

| 기술 | 용도 |
|------|------|
| Vanilla JavaScript | 화면 로직 (ES2020+, async/await) |
| HTML5 | 페이지 마크업 |
| CSS3 | 스타일링 (forma.css) |
| Spring Boot 정적 리소스 | 파일 서빙 (`/static/` 디렉토리) |

## 사용 금지 기술

| 금지 항목 | 사유 |
|-----------|------|
| React / Vue / Angular | 빌드 도구 없이 정적 파일만 사용 |
| Webpack / Vite / esbuild | 빌드 프로세스 없음 |
| jQuery / Bootstrap | 자체 컴포넌트(FormaGrid 등) 사용 |
| npm / yarn 패키지 | 외부 UI 라이브러리 금지 |

## 파일 배치 구조

```
src/main/resources/static/
├── forma/                  # 프레임워크 공통 (수정 거의 없음)
│   ├── forma.js            #   모든 공통 컴포넌트 포함
│   └── forma.css           #   ERP 업무 화면 스타일
├── index.html              # 메인 페이지
└── pages/                  # 화면별 (AI 생성 영역)
    ├── {모듈}/
    │   ├── {화면ID}.html   #   예: CUS-001.html
    │   └── {화면ID}.js     #   예: CUS-001.js
```

- HTML/JS는 `/static/` 아래에 직접 배치하며 빌드 과정이 없다
- 브라우저에서 `/pages/sales/ORD-001.html`로 직접 접근

## JavaScript 코딩 규칙

### page 객체 패턴

모든 화면 JS는 `page` 전역 객체에 상태와 메서드를 담는다.

```javascript
const page = {
    api: '/api/customer',

    init() {
        this.search = new FormaSearch('#search', [...], () => this.doSearch());
        this.grid = new FormaGrid('#grid', { columns: [...] });
        FormaPage.actionBar('#actions', [...]);
        this.doSearch();
    },

    async doSearch() {
        const data = await FormaApi.get(this.api, this.search.getValues());
        this.grid.setData(data.data, data.total);
    },
};

document.addEventListener('DOMContentLoaded', () => page.init());
```

### 규칙 요약

| 규칙 | 설명 |
|------|------|
| 전역 변수 최소화 | `page` 객체에 모든 것을 담는다 |
| async/await 사용 | Promise 체이닝(`.then()`) 금지 |
| DOM 조작 위임 | FormaGrid/FormaSearch 등 컴포넌트에 위임 |
| 직접 DOM 필요 시 | `document.querySelector()` 사용 |

## HTML 템플릿 구조

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>{화면명}</title>
    <link rel="stylesheet" href="/forma/forma.css">
</head>
<body>
<div class="forma-page">
    <h1 class="forma-page-title">{화면명}</h1>
    <div id="search"></div>
    <div id="actions"></div>
    <div id="grid"></div>
    <!-- master-detail 화면인 경우 -->
    <!-- <div id="detail-grid" style="margin-top:8px"></div> -->
</div>
<script src="/forma/forma.js"></script>
<script src="/pages/{모듈}/{화면ID}.js"></script>
</body>
</html>
```

### 스크립트 로딩 순서
1. `forma.css` - head에서 로딩
2. `forma.js` - body 하단에서 로딩 (공통 컴포넌트)
3. `{화면ID}.js` - forma.js 다음에 로딩 (화면 로직)

## CSS 체계

### 폰트

기본 폰트: Pretendard → Apple System → sans-serif (13px)

### 클래스 네이밍

모든 CSS 클래스는 `forma-` 접두사를 사용한다.

```
forma-{컴포넌트}                  → forma-grid, forma-search
forma-{컴포넌트}-{요소}           → forma-grid-wrap, forma-search-row
forma-{컴포넌트}-{상태/변형}      → forma-btn-primary, forma-grid-empty
```

### 레이아웃

- 최대 너비: 1400px (중앙 정렬)
- 그리드 컬럼: 2열 (FormaForm)
- 검색바/그리드: 1px solid #e0e0e0 테두리, 4px 라운드
