# 데이터 권한 설계

## 테이블 구조

```
tb_dept           부서 (트리 구조, parent_code)
tb_role           역할 (ADMIN, MANAGER, USER, VIEWER)
tb_user_role      사용자-역할 매핑 (N:N)
tb_menu           메뉴 (트리 구조, pgm_id 연결)
tb_role_menu      역할-메뉴 권한 (버튼별 Y/N)
tb_data_auth      데이터 권한 (역할별 데이터 접근 범위)
```

## 권한 체계 (3계층)

### 1. 메뉴 권한 (tb_role_menu)
역할별로 메뉴 접근 + 버튼(조회/신규/저장/삭제/출력/업로드/초기화) 제어.

| 역할 | 거래처관리 | 수주관리 |
|------|----------|---------|
| ADMIN | 전체 | 전체 |
| MANAGER | 전체 | 전체 |
| USER | 조회/신규/저장/초기화 | 조회/초기화 |
| VIEWER | 조회만 | 조회만 |

### 2. 데이터 권한 (tb_data_auth)

| auth_type | 설명 | SQL 조건 |
|-----------|------|---------|
| ALL | 전체 데이터 | (없음) |
| DEPT | 본인 부서만 | `AND dept_code = #{_userDept}` |
| DEPT_SUB | 본인+하위 부서 | `AND dept_code IN (하위부서 목록)` |
| USER | 본인 데이터만 | `AND created_by = #{_userId}` |

### 3. 버튼 권한 (프론트 FormaToolbar)
PGM init API에서 pgmAuth를 받아서 FormaToolbar가 버튼을 표시/숨김.

## 구현 아키텍처

```
TokenInterceptor.preHandle()
  ↓ JWT 파싱 → LoginUserVo
  ↓ DataAuthService.buildAuthParams(user)
  ↓ DataAuthContext.set({_userId, _userDept, _deptList, _dataAuthType, _isAdmin})

FormaSqlSession.selectList/selectOne()
  ↓ DataAuthContext.get() → 파라미터 Map에 자동 주입
  ↓ MyBatis XML에서 <include refid="common.dataAuth"/> 로 WHERE 조건 적용

TokenInterceptor.afterCompletion()
  ↓ DataAuthContext.clear()
```

### 매퍼 XML 사용법

```xml
<select id="selectGrid1" parameterType="map" resultType="map">
    SELECT * FROM tb_order
    <where>
        <if test="keyword != null">AND order_no LIKE '%' || #{keyword} || '%'</if>
        <include refid="common.dataAuth"/>
    </where>
</select>
```

`<include refid="common.dataAuth"/>` 한 줄로 데이터 권한 필터 적용.

## 부서 트리 해석

DataAuthService가 Java에서 전체 부서를 조회한 후 재귀적으로 하위 부서를 탐색.
5분 TTL 캐시로 성능 확보.

```
D00 (대표이사)
├── D10 (경영지원본부)
│   ├── D11 (인사팀)
│   └── D12 (재무팀)
├── D20 (기술본부)
│   ├── D21 (개발1팀)
│   └── D22 (개발2팀)
└── D30 (영업본부)
    ├── D31 (국내영업팀)
    └── D32 (해외영업팀)
```

park(영업본부, MANAGER, DEPT_SUB) → D30, D31, D32 데이터 접근 가능.
