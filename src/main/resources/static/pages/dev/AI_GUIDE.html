1

자연어로 화면 요구사항 입력

Claude에게 만들고 싶은 화면을 자연어로 설명합니다.

User > 품목관리 화면 만들어줘. 품목코드, 품목명, 규격, 단위, 단가 관리. 품목그룹별 검색. 단위는 코드 선택.
2

AI가 YAML 설계서 + DDL 생성

AI가 design/_prompts/00-generate-design.md 프롬프트를 참조하여 설계서를 생성합니다.

출력: design/screens/MMA010.yml + DDL + 코드 데이터

3

사용자가 설계서 검토/보완

YAML 파일을 열어 필드 추가/삭제, 위젯 변경, 권한 조정을 합니다.

설계서는 사람이 읽고 수정할 수 있는 중간 산출물입니다. AI가 100% 완벽하게 생성하지 않아도 됩니다.
4

확정된 설계서로 코드 자동 생성

AI가 design/_prompts/01-generate-code.md를 참조하여 4개 파일을 생성합니다.

# 생성되는 파일 src/main/java/com/forma/domain/material/item/Mma010Controller.java src/main/java/com/forma/domain/material/item/Mma010Service.java src/main/resources/mapper/domain/material/mma010.xml src/main/resources/static/pages/material/MMA010.html
5

SQL 보완 후 완성

생성된 SQL 초안을 검토하고, 복잡한 쿼리/비즈니스 로직을 보완합니다.

실행 방법

방법 1: Claude에게 직접 요청

User > 품목관리 화면 만들어줘. 품목코드, 품목명, 규격, 단위, 단가. 품목그룹별 검색. 단위는 코드 선택. # Claude가 자동으로: # 1) 00-generate-design.md 참조 → YAML 설계서 생성 # 2) 사용자 검토 요청 # 3) 확인 후 01-generate-code.md 참조 → 코드 생성

방법 2: 설계서부터 단계별 진행

# Step 1: 설계서 생성 User > design/_prompts/00-generate-design.md 참조해서 재고관리 화면 설계서 만들어줘 # Step 2: 검토 후 코드 생성 User > design/screens/MMA020.yml 설계서로 design/_prompts/01-generate-code.md 참조해서 코드 생성해줘

방법 3: 기존 화면 복제 후 수정

User > SDA010(거래처관리)을 참고해서 공급사관리 화면 만들어줘. 필드는 비슷한데 납품 이력 탭이 추가로 필요해.
